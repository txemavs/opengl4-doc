function `glDepthRange`
-----------------------

.. source file man4/glDepthRange

.. currentmodule:: OpenGL.GL

Specify mapping of depth values from normalized device coordinates to window coordinates

.. c:function:: glDepthRange
.. c:function:: void glDepthRange(GLdouble  nearVal, GLdouble  farVal, GLfloat  nearVal, GLfloat  farVal)
   :noindex: 


   
   :param nearVal:
      Specifies the mapping of the near clipping plane to window coordinates. The initial value is 0.
   
   :param farVal:
      Specifies the mapping of the far clipping plane to window coordinates. The initial value is 1.
   

.. seealso::

   
   :c:func:`glDepthFunc`, :c:func:`glPolygonOffset`, :c:func:`glViewport`, :c:func:`removedTypes`
   
.. rubric:: Description


After clipping and division by *`w`* , depth coordinates range from |EQUATION_1_| to 1, corresponding to the near and far clipping planes. :c:func:`glDepthRange` specifies a linear mapping of the normalized depth coordinates in this range to window depth coordinates. Regardless of the actual depth buffer implementation, window coordinate depth values are treated as though they range from 0 through 1 (like color components). Thus, the values accepted by :c:func:`glDepthRange` are both clamped to this range before they are accepted.

The setting of (0,1) maps the near plane to 0 and the far plane to 1. With this mapping, the depth buffer range is fully utilized.


**Associated gets:**


   * :c:func:`glGet` with argument ``GL_DEPTH_RANGE``



.. note::

   * It is not necessary that *nearVal* be less than *farVal* . Reverse mappings such as |EQUATION_2_| , and |EQUATION_3_| are acceptable.
   * The type of the *nearVal* and *farVal* parameters was changed from GLclampf to GLfloat for :c:func:`glDepthRangef` and from GLclampd to GLdouble for :c:func:`glDepthRange`. This change is transparent to user code and is described in detail on the :c:func:`removedTypes` page.
