function `glBlendFuncSeparate`
------------------------------

.. source file man4/glBlendFuncSeparate

.. currentmodule:: OpenGL.GL

Specify pixel arithmetic for RGB and alpha components separately

.. c:function:: glBlendFuncSeparate
.. c:function:: void glBlendFuncSeparate(GLenum  srcRGB, GLenum  dstRGB, GLenum  srcAlpha, GLenum  dstAlpha, GLuint  buf, GLenum  srcRGB, GLenum  dstRGB, GLenum  srcAlpha, GLenum  dstAlpha)
   :noindex: 


   
   :param buf:
      For :c:func:`glBlendFuncSeparatei`, specifies the index of the draw buffer for which to set the blend functions.
   
   :param srcRGB:
      Specifies how the red, green, and blue blending factors are computed. The initial value is ``GL_ONE``.
   
   :param dstRGB:
      Specifies how the red, green, and blue destination blending factors are computed. The initial value is ``GL_ZERO``.
   
   :param srcAlpha:
      Specified how the alpha source blending factor is computed. The initial value is ``GL_ONE``.
   
   :param dstAlpha:
      Specified how the alpha destination blending factor is computed. The initial value is ``GL_ZERO``.

.. rubric:: Description


Pixels can be drawn using a function that blends the incoming (source) RGBA values with the RGBA values that are already in the frame buffer (the destination values). Blending is initially disabled. Use :c:func:`glEnable` and :c:func:`glDisable` with argument ``GL_BLEND`` to enable and disable blending.

:c:func:`glBlendFuncSeparate` defines the operation of blending for all draw buffers when it is enabled. :c:func:`glBlendFuncSeparatei` defines the operation of blending for a single draw buffer specified by *buf* when enabled for that draw buffer. *srcRGB* specifies which method is used to scale the source RGB-color components. *dstRGB* specifies which method is used to scale the destination RGB-color components. Likewise, *srcAlpha* specifies which method is used to scale the source alpha color component, and *dstAlpha* specifies which method is used to scale the destination alpha component. The possible methods are described in the following table. Each method defines four scale factors, one each for red, green, blue, and alpha.

**Associated gets:**


   * :c:func:`glGet` with argument ``GL_BLEND_SRC_RGB``
   * :c:func:`glGet` with argument ``GL_BLEND_SRC_ALPHA``
   * :c:func:`glGet` with argument ``GL_BLEND_DST_RGB``
   * :c:func:`glGet` with argument ``GL_BLEND_DST_ALPHA``
   * :c:func:`glIsEnabled` with argument ``GL_BLEND``

**Errors:**

   * ``GL_INVALID_ENUM`` is generated if either *srcRGB* or *dstRGB* is not an accepted value.
   * ``GL_INVALID_VALUE`` is generated by :c:func:`glBlendFuncSeparatei` if *buf* is greater than or equal to the value of ``GL_MAX_DRAW_BUFFERS``.



.. note::

   * Incoming (source) alpha is correctly thought of as a material opacity, ranging from 1.0 ( |EQUATION_22_| ), representing complete opacity, to 0.0 (0), representing complete transparency.
   * When more than one color buffer is enabled for drawing, the GL performs blending separately for each enabled buffer, using the contents of that buffer for destination color. (See :c:func:`glDrawBuffer`.)
   * When dual source blending is enabled (i.e., one of the blend factors requiring the second color input is used), the maximum number of enabled draw buffers is given by ``GL_MAX_DUAL_SOURCE_DRAW_BUFFERS``, which may be lower than ``GL_MAX_DRAW_BUFFERS``.
