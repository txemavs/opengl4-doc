function `glBlendEquationSeparate`
----------------------------------

.. source file man4/glBlendEquationSeparate

.. currentmodule:: OpenGL.GL

Set the RGB blend equation and the alpha blend equation separately

.. c:function:: glBlendEquationSeparate
.. c:function:: void glBlendEquationSeparate(GLenum  modeRGB, GLenum  modeAlpha, GLuint  buf, GLenum  modeRGB, GLenum  modeAlpha)
   :noindex: 


   
   :param buf:
      for :c:func:`glBlendEquationSeparatei`, specifies the index of the draw buffer for which to set the blend equations.
   
   :param modeRGB:
      specifies the RGB blend equation, how the red, green, and blue components of the source and destination colors are combined. It must be ``GL_FUNC_ADD``, ``GL_FUNC_SUBTRACT``, ``GL_FUNC_REVERSE_SUBTRACT``, ``GL_MIN``, ``GL_MAX``.
   
   :param modeAlpha:
      specifies the alpha blend equation, how the alpha component of the source and destination colors are combined. It must be ``GL_FUNC_ADD``, ``GL_FUNC_SUBTRACT``, ``GL_FUNC_REVERSE_SUBTRACT``, ``GL_MIN``, ``GL_MAX``.
   

.. seealso::

   
   :c:func:`glGetString`, :c:func:`glBlendColor`, :c:func:`glBlendFunc`, :c:func:`glBlendFuncSeparate`
   
.. rubric:: Description


The blend equations determines how a new pixel (the ''source'' color) is combined with a pixel already in the framebuffer (the ''destination'' color). These functions specifie one blend equation for the RGB-color components and one blend equation for the alpha component. :c:func:`glBlendEquationSeparatei` specifies the blend equations for a single draw buffer whereas :c:func:`glBlendEquationSeparate` sets the blend equations for all draw buffers.

The blend equations use the source and destination blend factors specified by either :c:func:`glBlendFunc` or :c:func:`glBlendFuncSeparate`. See :c:func:`glBlendFunc` or :c:func:`glBlendFuncSeparate` for a description of the various blend factors.

In the equations that follow, source and destination color components are referred to as |EQUATION_1_| and |EQUATION_2_| , respectively. The result color is referred to as |EQUATION_3_| . The source and destination blend factors are denoted |EQUATION_4_| and |EQUATION_5_| , respectively. For these equations all color components are understood to have values in the range |EQUATION_6_| .

The results of these equations are clamped to the range |EQUATION_27_| .

The ``GL_MIN`` and ``GL_MAX`` equations are useful for applications that analyze image data (image thresholding against a constant color, for example). The ``GL_FUNC_ADD`` equation is useful for antialiasing and transparency, among other things.

Initially, both the RGB blend equation and the alpha blend equation are set to ``GL_FUNC_ADD``.




**Associated gets:**


   * :c:func:`glGet` with an argument of ``GL_BLEND_EQUATION_RGB``
   * :c:func:`glGet` with an argument of ``GL_BLEND_EQUATION_ALPHA``

**Errors:**

   * ``GL_INVALID_ENUM`` is generated if either *modeRGB* or *modeAlpha* is not one of ``GL_FUNC_ADD``, ``GL_FUNC_SUBTRACT``, ``GL_FUNC_REVERSE_SUBTRACT``, ``GL_MAX``, or ``GL_MIN``.
   * ``GL_INVALID_VALUE`` is generated by :c:func:`glBlendEquationSeparatei` if *buf* is greater than or equal to the value of ``GL_MAX_DRAW_BUFFERS``.



.. note::

   * The ``GL_MIN``, and ``GL_MAX`` equations do not use the source or destination factors, only the source and destination colors.
